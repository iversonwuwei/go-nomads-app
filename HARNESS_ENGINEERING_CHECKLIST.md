# Harness Engineering Checklist

本工程默认采用 Harness Engineering 范式，重点要求如下：

- 先明确边界：页面流程、路由、登录态、接口依赖、平台限制。
- 先补闭环：功能、加载态、错误态、空态、重试与失败恢复一起交付。
- 先保契约：接口字段、会员状态、支付状态、推送或实时连接变更要同步核对调用链。
- 先保可观测：关键链路需有日志或状态可追踪，能定位到页面、接口和用户操作。
- 先做验证：至少完成 build/test 或关键设备场景验证，并明确未验证机型风险。
- 先看回滚：避免把状态迁移、支付行为和 UI 语义改动混成一次不可控发布。
